# it-happened - Coding Agent Instructions

## Current Plan

**Active Feature**: [Documentation Improvement](specs/001-doc-improvement/plan.md)

## Project Overview

**it-happened** is an event management library written in Go that simplifies event-driven application development.

It provides clean abstractions for event creation, publishing, subscription, and handling with a focus on composability.

## Core Principles (Constitution)

All development MUST comply with the project constitution at `.specify/memory/constitution.md`.

**Full Constitution**: `.specify/memory/constitution.md`

## Technologies

- **Language**: Go 1.25+
- **Module**: `github.com/thomas-marquis/it-happened`
- **Testing**: 
  - `testify/assert` and `testify/require` for assertions
  - `go.uber.org/mock/mockgen` (gomock) for mock generation
  - Test structure: `t.Run()` subtests with Given/When/Then comments

## Project Structure

```
.
├── event/           # Core event types and interfaces (Bus, Event, Payload, Subscriber)
├── carrier/         # Event carrier implementations for batch event dispatching
├── inmemory/        # In-memory bus implementation
├── eventest/        # Testing utilities for events (coming soon...)
├── examples/        # Runnable examples demonstrating features (different from Go's example testing). Each subdirectory is an example with a `main` package and a `main()` function
├── docs/            # Project documentation (managed with MKdoc Material)
├── tools/           # Development utilities (lint.sh)
├── specs/           # Specification documents
├── go.mod           # Go module definition
├── go.sum           # Go dependencies checksum
├── gen.go           # Code generation configuration: must contains all the directives used by `go generate` (gomock, etc.)
├── Makefile         # Running repo management utilities
├── CONTRIBUTE.md    # Contribution guidelines
├── mkdocs.yml       # config file for MKdoc Material
└── README.md        # Project overview to help users getting started
```

## Development Workflow

1. **Test-First**: Write tests before implementation
2. **Mock Generation**: Use `go generate ./...` to generate/update mocks in internal/mocks/ directory
3. **Testing**: Run `go test ./...` to execute all tests
4. **Linting**: Run `./tools/lint.sh` before committing
5. **CI**: All GitHub Actions workflows must pass

## Definition of Done (from CONTRIBUTE.md)

A contribution is complete when:
- Implementation is clean, documented, and follows existing style
- Unit tests are added and all tests pass
- Lint script passes without errors
- Relevant documentation in docs/ is updated.
- All CI workflows pass
- Example added to examples/ for new features
- README.md updated if change is significant
- Constitution and other guideline files updated if needed

## Command References

- Generate mocks: `go generate ./...`
- Run tests: `go test ./...`
- Run lint: `./tools/lint.sh`
- Build: `go build ./...`
- Install: `go get github.com/thomas-marquis/it-happened`

## Important Notes

- DO NOT edit files in mocks/ manually - they are generated by mockgen
- Event IDs must be unique UUIDs
- ChainRef enables event correlation across distributed systems and chained events
- All public APIs must have Go doc comments
- Follow Go naming conventions (camelCase, PascalCase for exported names)
